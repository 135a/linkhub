  serverTimezone=Asia/Shanghai&

## 2. Redis 缓存配置

### 2.1 基础信息

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **镜像版本** | redis:latest | 最新稳定版 |
| **容器名称** | shortlink-redis | Docker 容器名 |
| **端口映射** | 6379:6379 | 默认 Redis 端口 |
| **密码** | 无（可选配置） | 生产环境建议设置 |

### 2.2 内存配置（重要）

```bash
# Redis 启动命令
redis-server --maxmemory 384mb --maxmemory-policy allkeys-lru
```

| 参数 | 值 | 说明 |
|------|-----|------|
| **maxmemory** | 384MB | Redis 最大内存限制 |
| **maxmemory-policy** | allkeys-lru | 内存淘汰策略：最近最少使用 |
| **Docker 内存限制** | 512MB | 容器最大内存 |
| **Docker 预留内存** | 256MB | 容器最小保证内存 |

💡 **为什么这样配置？**
- Docker 限制 512MB，Redis 自身占用 + 数据结构开销
- 设置 maxmemory 为 384MB（约 75%），预留空间给 Redis 内部结构
- allkeys-lru 策略确保内存不足时自动淘汰旧数据

### 2.3 应用场景

1. **短链接跳转缓存**
   - Key: `short-link:goto:{fullShortUrl}`
   - TTL: 根据短链接有效期动态设置

2. **空值缓存（防穿透）**
   - Key: `short-link:is-null:goto_{fullShortUrl}`
   - TTL: 30 分钟

3. **分布式锁**
   - Key: `short-link:lock:goto:{fullShortUrl}`
   - 用于缓存击穿防护

4. **流量控制**
   - 用户操作频率限制
   - Lua 脚本实现滑动窗口

5. **消息幂等性**
   - RocketMQ 消费去重

## 3. 布隆过滤器配置

### 3.1 Project 服务配置

**文件位置：** `project/src/main/java/com/nym/shortlink/project/config/RBloomFilterConfiguration.java`

```java
@Bean
public RBloomFilter<String> shortUriCreateCachePenetrationBloomFilter(RedissonClient redissonClient) {
    RBloomFilter<String> cachePenetrationBloomFilter = 
        redissonClient.getBloomFilter("shortUriCreateCachePenetrationBloomFilter");
    
    // 预期插入量：1 亿
    // 误判率：0.001 (0.1%)
    cachePenetrationBloomFilter.tryInit(100000000L, 0.001);
    
    return cachePenetrationBloomFilter;
}
```

| 参数 | 值 | 说明 |
|------|-----|------|
| **过滤器名称** | shortUriCreateCachePenetrationBloomFilter | Redis Key |
| **预期插入量** | 100,000,000 (1 亿) | 预计存储的短链接数量 |
| **误判率** | 0.001 (0.1%) | 可接受的误判概率 |
| **预估内存** | ~238 MB | 根据公式计算 |

### 3.2 Admin 服务配置

**文件位置：** `admin/src/main/java/com/nym/shortlink/admin/config/RBloomFilterConfiguration.java`

#### 用户注册布隆过滤器
```java

    RBloomFilter<String> bloomFilter =

    return bloomFilter;
}
```

| 参数 | 值 |
|------|-----|
| **过滤器名称** | userRegisterCachePenetrationBloomFilter |
| **预期插入量** | 1 亿 |
| **误判率** | 0.001 |

#### 分组标识布隆过滤器
```java

    RBloomFilter<String> bloomFilter =

```

| 参数 | 值 |
|------|-----|
| **过滤器名称** | gidRegisterCachePenetrationBloomFilter |
| **预期插入量** | 2 亿 |
| **误判率** | 0.001 |

### 3.3 自动恢复机制

✅ **配置冲突自动修复：**
```java
try {
    if (!bloomFilter.isExists()) {
        bloomFilter.tryInit(EXPECTED_INSERTIONS, FALSE_PROBABILITY);
    }
} catch (Exception e) {
    if (e.getMessage().contains("Bloom filter config has been changed")) {
        // 自动删除旧数据并重新初始化
        deleteAndRecreateBloomFilter(bloomFilter, redissonClient);
    }
}
```

**工作原理：**
1. 检测布隆过滤器是否存在
2. 不存在则初始化
3. 如果配置冲突（Redis 中已有不同配置的过滤器）
4. 自动删除旧数据并重新创建

### 3.4 作用说明

| 布隆过滤器 | 用途 | 拦截场景 |
|-----------|------|---------|
| shortUriCreateCachePenetrationBloomFilter | 短链接创建 | 防止重复创建相同短链接 |
| userRegisterCachePenetrationBloomFilter | 用户注册 | 防止用户名重复查询数据库 |
| gidRegisterCachePenetrationBloomFilter | 分组创建 | 防止分组标识重复查询 |

## 5. RocketMQ 消息队列

### 5.1 NameServer 配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **镜像版本** | apache/rocketmq:4.9.4 | RocketMQ 4.x |
| **容器名称** | rmqnamesrv | NameServer 容器 |
| **端口** | 9876 | NameServer 端口 |

**JVM 配置：**
```bash
JAVA_OPT=-Xms64m -Xmx128m -Xmn64m -XX:MetaspaceSize=32m -XX:MaxMetaspaceSize=64m
```

**资源限制：**
- 内存限制：256MB
- 内存预留：128MB

### 5.2 Broker 配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **容器名称** | rmqbroker | Broker 容器 |
| **端口** | 10909, 10911 | Broker 通信端口 |
| **NameServer 地址** | rocketmq-namesrv:9876 | 服务发现 |

**JVM 配置：**
```bash
JAVA_OPT=-Duser.home=/opt \
  -Xms512m -Xmx1024m -Xmn512m \
  -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=256m
```

**资源限制：**
- 内存限制：1536MB (1.5GB)
- 内存预留：1GB

### 5.3 Producer 配置（Project 服务）

```yaml
rocketmq:
  name-server: ${ROCKETMQ_NAME_SERVER:127.0.0.1:9876}
  producer:
    group: short-link_project-service_stats-save_pg
    topic: short-link_project-service_topic
    send-message-timeout: 2000          # 发送超时 2 秒
    retry-times-when-send-failed: 1     # 同步发送失败重试 1 次
    retry-times-when-send-async-failed: 1  # 异步发送失败重试 1 次
```

### 5.4 Consumer 配置

```yaml
rocketmq:
  consumer:
    group: short-link_project-service_stats-save_cg
```

### 5.5 应用场景

✅ **短链接访问统计削峰填谷：**
1. 用户访问短链接
2. 统计数据封装为消息
3. 发送到 RocketMQ
4. 消费者异步处理，批量写入数据库
5. 避免高频写操作冲击数据库

## 6. ShardingSphere 分库分表

### 6.1 驱动配置

```yaml
spring:
  datasource:
    driver-class-name: org.apache.shardingsphere.driver.ShardingSphereDriver
    url: jdbc:shardingsphere:classpath:shardingsphere-config-${database.env:dev}.yaml
```

### 6.2 分表策略（Admin 服务）

**配置文件：** `admin/src/main/resources/shardingsphere-config-dev.yaml`

#### t_user 表分片
```yaml
tables:
  t_user:
    actualDataNodes: ds_0.t_user_${0..15}  # 16 张表
    tableStrategy:
      standard:
        shardingColumn: username            # 分片键
        shardingAlgorithmName: user_table_hash_mod
```

#### t_group 表分片
```yaml
tables:
  t_group:
    actualDataNodes: ds_0.t_group_${0..15}  # 16 张表
    tableStrategy:
      standard:
        shardingColumn: username             # 分片键
        shardingAlgorithmName: group_table_hash_mod
```

#### 分片算法
```yaml
shardingAlgorithms:
  user_table_hash_mod:
    type: HASH_MOD
    props:
      sharding-count: 16  # 16 个分片
  group_table_hash_mod:
    type: HASH_MOD
    props:
      sharding-count: 16
```

### 6.3 数据加密

```yaml
rules:
  - !ENCRYPT
    tables:
      t_user:
        columns:
          phone:
            cipherColumn: phone
            encryptorName: common_encryptor
          mail:
            cipherColumn: mail
            encryptorName: common_encryptor
        queryWithCipherColumn: true
    encryptors:
      common_encryptor:
        type: AES
        props:
          aes-key-value: d6oadClrrb9A3GWo
```

**加密字段：**
- 手机号（phone）
- 邮箱（mail）

**加密算法：** AES

## 🔧 常见问题与解决方案

### 1. Redis 布隆过滤器配置冲突

**错误信息：**
```
ERR user_script:1: Bloom filter config has been changed
```

**原因：** Redis 中已存在不同配置的布隆过滤器

**解决方案：**
代码已实现自动恢复机制，会自动删除旧数据并重新初始化。

### 2. MySQL 远程连接失败

**解决方案：**
```sql
-- 授权远程主机连接
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'root' WITH GRANT OPTION;
FLUSH PRIVILEGES;
```

### 3. Nacos 配置不生效

**原因：** 本地环境变量覆盖了 Nacos 配置

**解决方案：**
```bash
# 设置环境变量
export SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR=127.0.0.1:8848
export SPRING_DATA_REDIS_HOST=127.0.0.1
```

### 4. Docker 配置修改后未生效

**解决方案：**
```bash
# 强制重建镜像和容器
docker-compose down
docker-compose up --build -d
```

### 5. RocketMQ 消息积压

**监控指标：**
- Broker 内存使用率
- Consumer 消费速度
- 消息堆积数量

**优化方案：**
- 增加 Consumer 实例数量
- 优化消费逻辑性能
- 调整 Broker JVM 参数
