## 新增需求

### 需求：Java 服务全局 AOP 日志
自动记录 Java 模块中 Controller 层和 Service 层所有方法的执行情况。

#### 场景：方法进入和退出日志
- **当 (WHEN)** 执行匹配 `com.nym.shortlink.*.controller..*` 或 `com.nym.shortlink.*.service..*` 包路径的任何方法时。
- **那么 (THEN)** 以 `DEBUG` 级别记录方法名称、入参（进入时）以及返回值/执行耗时（退出时）。

#### 场景：异常日志记录
- **当 (WHEN)** 任何方法内发生异常时。
- **那么 (THEN)** 以 `ERROR` 级别记录异常消息和完整堆栈跟踪。

### 需求：Go 网关 Gin 中间件日志
实现中间件以记录 Go 网关处理的所有 HTTP 请求。

#### 场景：请求/响应日志记录
- **当 (WHEN)** 接收到 HTTP 请求时。
- **那么 (THEN)** 以 `DEBUG` 级别记录请求方法、路径、IP 和 Header，并以 `INFO` 级别记录响应状态码和耗时。

### 需求：敏感数据脱敏
确保日志中不会记录明文形式的敏感信息。

#### 场景：密码脱敏
- **当 (WHEN)** 日志条目包含名为 `password`、`token` 或 `secret` 的字段时。
- **那么 (THEN)** 必须使用 `******` 遮盖其值。

### 需求：环境特定的日志级别
日志级别必须可通过外部配置。

#### 场景：开发环境日志
- **当 (WHEN)** 激活的 profile 为 `dev` 时。
- **那么 (THEN)** 默认日志级别应为 `DEBUG`。

#### 场景：生产环境日志
- **当 (WHEN)** 激活的 profile 为 `prod` 时。
- **那么 (THEN)** 默认日志级别应为 `INFO`，并优先记录 `ERROR` 和 `WARN`。
