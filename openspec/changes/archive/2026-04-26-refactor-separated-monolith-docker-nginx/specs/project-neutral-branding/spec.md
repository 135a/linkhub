## ADDED Requirements

### Requirement: 项目对外文案必须去营销化并保持中性专业
系统与文档中的对外文案 MUST 移除与个人营销强绑定的表达（例如“拿个Offer”“马丁”），统一为中性、工程导向、可审阅的专业描述。

#### Scenario: 前端页面文案合规
- **WHEN** 用户浏览首页、功能页与页脚等公共区域
- **THEN** 页面 SHALL 不出现被定义为营销化的禁用词，并呈现中性项目说明

#### Scenario: 文档文案合规
- **WHEN** 评审者阅读 README 与部署文档
- **THEN** 文档 MUST 使用技术事实与工程目标描述项目，不包含营销口号式内容

### Requirement: 文案治理必须可复检
系统 MUST 提供可重复执行的关键词检查流程，以避免后续迭代引入不符合定位的文案。

#### Scenario: 提交前执行文案检查
- **WHEN** 开发者在提交变更前运行内容检查脚本
- **THEN** 脚本 SHALL 输出命中项并阻止包含禁用词的变更进入主分支

#### Scenario: 新增页面自动纳入检查范围
- **WHEN** 新增前端页面或文档文件
- **THEN** 检查流程 MUST 默认覆盖新文件而无需手工维护路径白名单
