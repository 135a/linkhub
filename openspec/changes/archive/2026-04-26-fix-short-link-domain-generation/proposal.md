## 背景

目前，在创建短链接时，系统会忽略请求中的 `domain` 字段，始终使用配置文件中的默认域名（`127.0.0.1:8001`）。这导致生成的完整短链接（`fullShortUrl`）不符合预期，即使请求中指定了自定义域名（如 `shortlink.nym.asia`），生成的链接仍然是本地回环地址。

## 修改内容

- 更新 `ShortLinkServiceImpl` 中的短链接创建逻辑，优先使用请求参数中的 `domain` 字段。
- 确保在构造 `fullShortUrl` 时使用正确的域名。
- 确保布隆过滤器校验、数据库插入以及 Redis 缓存中使用的完整短链接均使用指定的域名。
- 将系统默认域名配置改为从环境变量 `DOMAIN` 获取，以支持不同环境下的域名动态切换。
- 更新前端 Vuex store，使其从环境变量 `VITE_DOMAIN` 获取域名，消除硬编码。


## 功能点

### 新功能
- 无

### 修改功能
- `short-link-creation`: 支持在创建短链接时指定自定义域名，若未指定则回退到默认域名。

## 影响范围

- `ShortLinkServiceImpl.java` (创建短链接逻辑)
- 短链接创建相关的 API 接口。
