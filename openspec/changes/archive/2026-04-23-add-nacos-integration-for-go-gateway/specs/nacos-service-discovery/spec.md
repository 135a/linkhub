# Nacos 服务发现

## 需求

系统应从 Nacos 动态获取后端服务地址，以支持服务动态扩缩容和迁移。

## 功能规格

### 1. 服务发现

**系统应**在启动时连接 Nacos 并获取已注册的服务实例列表。

**系统应**根据路由配置中的 `service_name` 查询对应的服务实例。

**系统应**支持多个实例的负载均衡（简单轮询）。

### 2. 降级策略

**系统应**在 Nacos 连接失败时，使用 `config.yaml` 中配置的 `default_target` 作为后端地址。

**系统应**在服务未注册到 Nacos 时，使用 `default_target` 作为后端地址。

**系统应**在降级时记录 WARNING 级别日志，包含降级原因。

### 3. 定时刷新

**系统应**每 30 秒（可配置）从 Nacos 刷新一次服务列表。

**系统应**在刷新时使用新列表替换旧缓存。

**系统应**在刷新失败时保留旧缓存，不降级。

### 4. 健康检查

**系统应**在查询服务实例时过滤不健康的实例。

**系统应**优先返回健康的实例。
