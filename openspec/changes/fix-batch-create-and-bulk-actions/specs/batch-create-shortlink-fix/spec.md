## ADDED Requirements

### Requirement: 批量创建短链接必须实际创建并返回 JSON 结果
前端调用批量创建接口（`POST /create/batch`）时，SHALL 使用标准 JSON 响应模式，禁止设置 `responseType: 'arraybuffer'`。接口调用成功后，前端 SHALL 弹出成功提示并刷新短链接列表，不得触发任何文件下载行为。

#### Scenario: 批量创建成功后显示提示并刷新列表
- **WHEN** 用户填写跳转链接和描述信息后点击「确认」
- **THEN** 系统 SHALL 调用 `POST /create/batch` 并以 JSON 格式接收响应，弹出「创建成功」提示，关闭对话框，主页面列表自动刷新以展示新创建的短链接

#### Scenario: 批量创建失败时显示错误信息
- **WHEN** 后端返回非成功响应（`success: false` 或网络错误）
- **THEN** 前端 SHALL 弹出包含错误信息的提示，对话框保持打开状态，不触发文件下载

### Requirement: 批量创建时描述信息必须自动从标题接口获取
当用户在批量创建对话框中输入跳转链接后，系统 SHALL 对每一行有效链接自动调用 `GET /title` 接口查询网页标题，并按行顺序填充到描述信息文本框中。用户可在自动填充完成后手动修改任意行的描述信息。

#### Scenario: 输入链接后自动填充描述信息
- **WHEN** 用户在跳转链接文本框中输入若干行有效 URL 并触发失焦或等待防抖
- **THEN** 系统 SHALL 依次对每行 URL 调用 `GET /title` 接口，将返回的标题按原始行顺序填充到描述信息文本框，描述信息行数与跳转链接行数始终相等

#### Scenario: 标题接口调用失败时使用空字符串占位
- **WHEN** 某行链接调用 `GET /title` 接口超时或返回错误
- **THEN** 该行描述信息 SHALL 填充为空字符串，其余行正常填充，不阻断整体流程

#### Scenario: 用户手动修改自动填充的描述信息
- **WHEN** 系统已自动填充描述信息后，用户对某行进行手动编辑
- **THEN** 系统 SHALL 保留用户的修改，不因链接内容未变化而覆盖用户输入

### Requirement: 批量创建描述信息与跳转链接行数必须严格对应
在提交批量创建请求前，前端 SHALL 校验描述信息的行数与跳转链接的行数完全相等，不允许多于或少于。

#### Scenario: 行数不一致时阻止提交
- **WHEN** 用户点击「确认」时描述信息行数与跳转链接行数不相等
- **THEN** 系统 SHALL 阻止提交并显示校验错误「描述信息行数与链接行数不等」
