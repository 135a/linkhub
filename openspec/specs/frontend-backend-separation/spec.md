# frontend-backend-separation

## Purpose
定义前后端解耦与契约集成规范，确保独立构建、独立验证与稳定协作演进。

## Requirements

### Requirement: 前后端职责边界清晰且通过契约集成
系统 MUST 将前端应用与单体后端 API 作为独立构建与运行单元，并通过版本化 HTTP API 契约进行集成；前端 MUST NOT 依赖网关路由规则、后端内部实现或数据库结构。

#### Scenario: 前端仅通过 API 访问业务数据
- **WHEN** 前端需要读取或写入短链相关业务数据
- **THEN** 前端 SHALL 仅调用约定的 API 端点并基于响应模型渲染，不直接访问后端私有模块或数据库

#### Scenario: 后端变更受契约约束
- **WHEN** 后端准备调整接口字段或语义
- **THEN** 变更 MUST 通过 API 契约版本控制与兼容性检查，避免未声明破坏前端集成

### Requirement: 前后端应支持独立构建与发布验证
系统 MUST 支持前端与单体后端分别执行构建、测试与打包流程，并在持续集成中对双方进行独立质量门禁；后端验证流程 MUST NOT 依赖 Nacos 或远程服务可达性。

#### Scenario: 前端构建失败不阻塞后端单元测试执行
- **WHEN** 前端构建任务失败
- **THEN** 流水线 SHALL 仍执行后端单元/集成测试并分别报告结果

#### Scenario: 后端契约测试失败阻止发布
- **WHEN** 后端变更导致契约测试失败
- **THEN** 发布流程 MUST 阻止该版本进入部署阶段并输出失败原因
